# 記事分析

指定された記事を詳細に分析し、改善提案を行ってください。

## 分析項目

### 1. SEO分析

- **タイトル分析**:
  - 文字数（推奨: 60文字以内）
  - キーワード配置（前半にメインキーワードがあるか）
  - 魅力度（数字・疑問形・具体性があるか）

- **メタディスクリプション分析**:
  - 文字数（推奨: 120-160文字）
  - キーワード含有
  - 読者メリットの明確さ

- **見出し構造分析**:
  - h1, h2, h3の階層構造
  - 見出しの数（h2: 3-5個推奨）
  - キーワードの自然な配置

- **内部リンク分析**:
  - 内部リンク数
  - アンカーテキストの適切性
  - リンク切れの有無

### 2. 可読性分析

- **段落構成**:
  - 段落の長さ（100-200文字推奨）
  - 改行の適切性

- **文章の読みやすさ**:
  - 一文の長さ
  - 専門用語の使用頻度
  - 箇条書き・リストの活用

- **視覚的要素**:
  - 画像の配置と数
  - 図解・表の使用
  - 強調表示の適切性

### 3. コンテンツ品質分析

- **文字数**: 800文字以上か
- **オリジナリティ**: 独自の視点・経験があるか
- **E-E-A-T**:
  - Experience（経験）: 体験談・実例
  - Expertise（専門性）: 技術的正確性
  - Authoritativeness（権威性）: 信頼できる情報源
  - Trustworthiness（信頼性）: メリット・デメリット両論

### 4. AdSense要件チェック

- コンテンツ要件
- 画像要件
- SEO要件
- 禁止コンテンツチェック

## 実行方法

1. WordPress REST APIで記事を取得
2. 各項目を分析
3. 改善提案をまとめる
