## AdSense審査を意識した記事作成の基本

### ✅ 読者価値を最優先

- **具体的な問題解決**: 「〜する方法」「〜の手順」など、実用的な内容
- **独自の視点**: 自分の経験・実例を含める
- **網羅性**: 初心者でも理解できるよう、前提知識から丁寧に説明

### ✅ E-E-A-Tを意識

- **Experience（経験）**: 実際に試した結果・体験談
- **Expertise（専門性）**: 技術的な正確性・深い知識
- **Authoritativeness（権威性）**: 信頼できる情報源
- **Trustworthiness（信頼性）**: 正直な評価・デメリットも記載

## 文字数・コンテンツ要件

### 最低基準

- **文字数**: 800文字以上（推奨: 1000-2000文字）
- **段落**: 1段落100-200文字程度
- **見出し**: h2見出し3-5個、必要に応じてh3も使用

### 質の高い記事の特徴

1. **導入で惹きつける**: 読者の課題を明確化
2. **具体例を豊富に**: コード例・スクリーンショット・図解
3. **段階的な説明**: ステップ・バイ・ステップで解説
4. **まとめで振り返る**: 要点を箇条書きで整理

### 避けるべき内容

❌ **コピペコンテンツ**: 他サイトからの転載は絶対NG
❌ **薄いコンテンツ**: 情報量が少なく、価値が低い記事
❌ **誤情報**: 未検証の情報・古い情報
❌ **禁止コンテンツ**: アダルト・暴力・違法・差別的内容

## 画像の使い方

### アイキャッチ画像

- **サイズ**: 最低1200×630px（16:9推奨）
- **形式**: JPEG / PNG
- **ソース**: Unsplash（無料・商用利用可）推奨
- **設定**: 記事公開後、`add_featured_images.py`で自動設定

### 本文画像

- **配置**: h2見出しの直後に1枚（計3枚以上）
- **目的**: 視覚的な理解を助ける、テキストの分割
- **種類**:
  - スクリーンショット（手順説明）
  - 図解（概念説明）
  - 関連イメージ（イメージ喚起）
- **自動挿入**: `apply_post_improvements.py`で自動追加可

### 画像の最適化

- **ファイルサイズ**: 200KB以下推奨（ページ速度対策）
- **altテキスト**: 画像の説明を必ず記載（SEO対策）
- **クレジット**: Unsplash使用時は自動で"Photo by [photographer] on Unsplash"が追加される

## SEO対策

### タイトルタグ

- **文字数**: 60文字以内
- **キーワード**: メインキーワードを前半に配置
- **魅力的**: 数字・疑問形・具体性を含める

**良い例**:
- ✅ 「Python初心者向け！環境構築の5つのステップを解説」
- ✅ 「副業で月5万円稼ぐ方法【2026年最新版】」

**悪い例**:
- ❌ 「Python環境構築」（短すぎる、魅力なし）
- ❌ 「プログラミングの環境構築をするための手順について詳しく解説していきます」（長すぎる）

### メタディスクリプション

- **文字数**: 120-160文字
- **内容**: 記事の要約、読者メリットを明確に
- **キーワード**: 自然に含める（詰め込みすぎない）
- **自動設定**: `apply_post_improvements.py`で自動生成

**良い例**:
```
Pythonの環境構築方法を初心者向けに解説します。Windows・Mac対応の手順を図解付きで説明。15分で完了する簡単セットアップです。
```

### 見出し構造

- **h1**: タイトル（1個のみ）
- **h2**: メインセクション（3-5個）
- **h3**: サブセクション（必要に応じて）

**良い例**:
```
h1: Pythonの環境構築手順【初心者向け】
  h2: Pythonとは？
  h2: Windows環境での構築手順
    h3: ステップ1: インストーラーのダウンロード
    h3: ステップ2: インストールの実行
  h2: Mac環境での構築手順
  h2: 動作確認の方法
  h2: まとめ
```

## 内部リンク戦略

### 自然な内部リンク

- **関連記事セクション**: 記事末尾に「関連記事」セクションを作成
- **本文内リンク**: 関連する話題が出たら自然にリンク
- **リンク数**: 2-5個程度（多すぎない）

### 効果的なアンカーテキスト

**良い例**:
- ✅ 「[Pythonの基礎文法について詳しく解説](リンク)」
- ✅ 「詳しくは[こちらの記事](リンク)で解説しています」

**悪い例**:
- ❌ 「[こちら](リンク)」（何のリンクかわからない）
- ❌ 「[クリック](リンク)」（SEO効果なし）

### 自動化ツール

- `improve_post_structure.py --mode suggest-links`: 関連記事を自動提案
- `apply_post_improvements.py`: 関連記事セクションを自動追加

## 避けるべきこと

### コンテンツ面

❌ **他サイトのコピペ**: 絶対にNG、AdSense審査で即落ち
❌ **薄い記事の量産**: 質より量は逆効果
❌ **キーワード詰め込み**: 不自然なキーワード多用
❌ **誇大広告**: 「絶対稼げる」「100%成功」など
❌ **禁止コンテンツ**: アダルト、ギャンブル、違法、暴力、差別

### 技術面

❌ **リンク切れ**: 定期的にチェック
❌ **低品質な画像**: ぼやけた画像、著作権違反
❌ **モバイル未対応**: レスポンシブデザイン必須
❌ **遅いページ速度**: 画像最適化、キャッシュ活用

### 運営面

❌ **長期間の更新停止**: 最低月2-4記事は更新
❌ **プライバシーポリシーなし**: 必須ページを設置
❌ **お問い合わせ手段なし**: 連絡先を明示

## Claude Codeでの記事作成フロー

Claude Codeを活用することで、AdSense対応記事を効率的に作成できます。

### ステップ1: 記事企画

**プロンプト例**:
```
「しかく猫の部屋」ブログ用に、[テーマ]に関する記事を企画したいです。
AdSense審査を意識した、以下の条件を満たす記事構成を提案してください：

- 文字数: 1000-1500文字
- h2見出し: 4-5個
- 読者ターゲット: [初心者/中級者/上級者]
- キーワード: [メインキーワード]

templates/post_template.md と templates/post_guidelines.md を参考にしてください。
```

### ステップ2: 記事執筆

**プロンプト例**:
```
以下の構成に基づいて、記事本文を執筆してください。
AdSenseガイドラインを遵守し、独自性と読者価値を重視してください。

[提案された構成を貼り付け]

注意点:
- E-E-A-Tを意識した内容
- 具体例・コード例を含める
- 800文字以上
- まとめセクションを含める
```

### ステップ3: WordPressへの投稿

1. 生成された記事をWordPress管理画面にコピー
2. カテゴリ・タグを設定
3. 公開

### ステップ4: 自動最適化

```bash
# アイキャッチ画像を自動設定
python scripts/active/add_featured_images.py

# 記事改善を一括適用（DRY RUNで確認）
python scripts/active/apply_post_improvements.py --mode all --dry-run

# 実行
python scripts/active/apply_post_improvements.py --mode all
```

### ステップ5: 確認

- 表示確認（PC・スマホ）
- リンク切れチェック
- 誤字脱字チェック
- メタディスクリプション確認

## まとめ

AdSense対応の記事作成は、以下の3つを意識することが重要です：

1. **読者価値**: 具体的な問題解決・独自の視点
2. **基準遵守**: 文字数・画像・SEO要件を満たす
3. **継続性**: 定期的な更新・品質の維持

Claude Codeと自動化ツールを活用することで、これらの要件を効率的に満たすことができます。
